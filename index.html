  1. O QUE É?

    • CodeIgniter é um poderoso framework PHP leve e simples.
    • É um kit de ferramentas para desenvolver aplicações web cheias de funcionalidades.
    • Contém um conjunto de bibliotecas para tarefas comuns necessárias (usadas como base para as aplicações), interfaces simples e uma estrutura lógica para acesso a estas bibliotecas.


  2. PARA QUE SERVE?

    • Esse “toolkit” permite facilitar o desenvolvimento de diversas aplicações, poupando tempo e custos para codificação.
    • Por ser um framework, é um otimizador de recursos que objetiva a resolução de problemas.
    • Assim, resolve problemas recorrentes com uma abordagem genérica, fazendo com que o desenvolvedor foque nos “problemas” da aplicação, não na arquitetura e configurações.
    • Com o CodeIgniter, são possíveis as implementações, por exemplo, da conexão com o banco de dados, tratamento e consultas de dados retornados, construção de formulários e outros conteúdos HTML para a criação da interface visual da aplicação, entre várias outras.


  3. QUEM USA?

    • Desenvolvedores PHP que precisam de um kit de ferramentas simples para criar aplicativos web completos.


  4. VANTAGENS E DESVANTAGENS;

      5. Vantagens:

    • É um framework simples, compatível com a maior quantidade de versões do software, o que proporciona sua instalação na maioria de servidores possíveis.
    • Conhecido pelo seu alto desempenho,composto por kit de bibliotecas básicas, fácil implementação para desenvolver aplicativos de todos os tamanhos.
    • A Manipulação de banco de dados com o uso dos principais Frameworks, possui uma facilidade maior na criação de query’s de banco de dados para consulta, inclusão, exclusão, etc.

      • Desvantagens:

    • Uma desvantagem que desqualifica muito o codeIgniter é suas atualizações, o que provoca uma perda de recursos para outros Frameworks PHP, e o baixo rendimento na segurança.


  5. QUAIS RECURSOS OFERECE?

    • Acelera o desenvolvimento web em PHP (incluindo hospedagem PHP) por conta de seu vasto arranjo de pacotes e bibliotecas. Essas bibliotecas são desenvolvidas para áreas do desenvolvimento de web onde a reutilização de códigos é comum, tirando das mãos dos desenvolvedores um bocado de trabalho.
    • Baseado na arquitetura modelo-visão-controlador (MVC), o CodeIgniter também adiciona uma camada de lógica para suas aplicações de web. Isso não só contribui para um design mais limpo, mas também é útil em paralelo com certos processos no ciclo de desenvolvimento. Resultando disso, seus produtos terão uma alta resposta à possibilidade de mudanças de mercado e negócios mais rapidamente.


  6. COMO INSTALAR E USAR?

    • Para utilizar o Framework, basta baixar o arquivo diretamente do http://codeigniter.com/, descompactá-lo dentro do diretório root do servidor e renomear a pasta para algum nome qualquer. Depois disso, o CodeIgniter criará uma estrutura simples inicial com uma página de exemplo, basta chamar http://localhost/NOME_DO_DIRETORIO e verá uma página com a tela inicial de boas vindas.
